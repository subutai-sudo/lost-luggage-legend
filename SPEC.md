# Lost Luggage Legends ‚Äî Technical Specification

## 1. Concept & Vision

**Lost Luggage Legends** is an AI-run niche travel media site focused on *carry-on essentials for business travelers*. The site is the product ‚Äî authoritative, specific, no-fluff gear guides and articles that rank and convert. The *system* behind it (AI content pipeline + QA + publish workflow) is the operational moat. The brand voice is confident, precise, and practical ‚Äî like a seasoned road warrior sharing hard-won insights.

## 2. Design Language

**Aesthetic:** Clean editorial ‚Äî think Wirecutter meetsafar. High information density, clear hierarchy, generous whitespace.
**Colors:**
- Primary: `#1a365d` (deep navy ‚Äî trust, authority)
- Accent: `#dd6b20` (burnt orange ‚Äî energy, CTA buttons)
- Background: `#fafaf9` (warm white)
- Text: `#1a1a1a` (near-black)
- Muted: `#6b7280` (secondary text)
**Typography:**
- Headings: `Inter`, 700 weight
- Body: `Inter`, 400 weight
- Mono (specs): `JetBrains Mono`
**Motion:** Minimal ‚Äî 200ms ease transitions on links/buttons, no animation on content load.

## 3. Stack

- **Framework:** Astro 5 (static-first, zero-JS by default, excellent SEO)
- **Content:** MDX (markdown + components)
- **Hosting:** Vercel (zero-config, generous free tier, edge network)
- **CI/CD:** GitHub Actions (deploy on push to `main`)
- **Analytics:** Plausible Analytics (privacy-first, no cookies, free tier up to 10k pageviews/mo)
- **AI Content:** OpenAI GPT-4o via `src/scripts/generate-article.mjs` CLI tool
- **Domain:** `lostluggagelegends.com` (to be registered ‚Äî DNS pointing to Vercel)

## 4. Site Structure

```
/                   ‚Äî Homepage (hero + featured articles + categories)
/about              ‚Äî About page (who we are, our methodology)
/articles/         ‚Äî Article index (filterable by category)
/articles/[slug]    ‚Äî Individual articles
/rss.xml            ‚Äî RSS feed
/sitemap-index.xml  ‚Äî Sitemap
```

**Categories:** `carry-on-bags`, `packing-essentials`, `tsa-rules`, `business-travel`, `gear-reviews`

## 5. Content Pipeline (Draft ‚Üí Publish)

```
1. Generate  ‚Üí src/scripts/generate-article.mjs (GPT-4o ‚Üí MDX draft)
2. Store    ‚Üí src/content/articles/[slug].mdx (draft flag)
3. Review   ‚Üí Human review in GitHub PR
4. QA       ‚Üí QA checklist (affiliate disclosure, alt-text, internal links, facts)
5. Publish  ‚Üí Merge PR ‚Üí GitHub Actions ‚Üí Vercel deploy
```

## 6. On-Page SEO (LOS-3)

- Meta title: `{Article Title} | Lost Luggage Legends`
- Meta description: 150-160 chars, keyword-first
- Open Graph + Twitter cards on every article
- JSON-LD schema: `Article` schema with author, datePublished, image
- Canonical URLs on all pages
- `robots.txt` allowing all crawlers
- Sitemap auto-generated by @astrojs/sitemap

## 7. Affiliate Compliance

- `AffiliateDisclosure` component on every article page
- Links use `rel="sponsored nofollow"` for paid content
- Amazon Associates compliant link structure

## 8. Analytics (Plausible)

- Script injected in `<head>` via BaseLayout
- Custom events: `article_read`, `affiliate_click`, `newsletter_signup`
- Dashboard: plausible.com/lostluggagelegends.com

## 9. GitHub Actions CI/CD

- Trigger: push to `main`
- Steps: install ‚Üí build ‚Üí deploy to Vercel
- Preview deployments for PRs
- Vercel token stored as GitHub Actions secret
